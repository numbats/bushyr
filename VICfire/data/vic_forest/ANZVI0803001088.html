Additional Metadata: 
                            
                              History: This layer was previously called VEG500 but renamed to a more appropriate name on 21 April, 1996 when the layer was documented and approved by the DDRG.  Custodianship of this layer has been transferred to the Forest Service as of 11/04/2000.

Relationship to other Datasets: The FORTYPE500 dataset will be superseded at the 1:100,000 by the SVEG100 dataset. There should be a broad consistency between the two datasets. 

The external forest boundary of the FORTYPE500 dataset is derived from the Forest Cover Changes in Victoria (FCOV500_7287) dataset which was derived from 1987 LANDSAT MSS imagery.   Some anomalies have been found in the boundary and require checking.  The FORTYPE500 is registered to the 1:500,000 topographic features.


Current Design Issues: The current layer design has been kept simple with only the one PAT item (FTYPE).  The FTYPE item is used to reference the dominant and two associated forest species, maximum and minimum mature stand height, foliage cover, and broad forest type category.

The current structure is designed to assist limited dataset updates and not display / query operations.


Future Design Issues: The current reference table that contains primary forest attributes should be directly linked to the coverage to allow unique polygon attribute editing and querying. The dominant and associated species should be referenced using the standard numeric species codes developed by the Flora Section, NRE and used in the Structural Vegetation layer (SVEG100).
